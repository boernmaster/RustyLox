Global Topic Filter

Configure a global regex filter to exclude unwanted MQTT messages from being sent to the Miniserver. The filter applies to all subscriptions.

Regular expression to filter MQTT messages globally. Matching messages are NOT forwarded to Miniserver.
Topic slashes (/) are replaced with underscores (_) before filtering.
Leave empty to disable filtering.
📝 Common Filter Examples

Exclude system topics:

_healthcheck_|_info_|_announce_

Exclude gateway/schedule:

_mqttgateway_|_schedule_

Exclude specific prefix:

^solcast_

Combine multiple patterns:

_healthcheck_|_info_|_announce_|_mqttgateway_|_schedule_|^solcast_

Quick add buttons:

ℹ️ How It Works
  • Topic home/sensor/_healthcheck_ becomes home_sensor__healthcheck_
  • Pattern _healthcheck_ matches → message is excluded
  • Pattern ^solcast_ matches topics starting with "solcast"
  • Use | to combine multiple patterns
